Stage 1:

Master Project Context 
App: Gmail inbox recovery tool — users sign in with Google, scan their inbox, classify senders as junk or safe, then bulk delete / unsubscribe / mark read / archive. Senders are shown as a list with checkboxes so the user can select exactly which senders to act on.

Stack:

Next.js 14+ (App Router, TypeScript)
Supabase (auth, database, token storage)
Vercel (hosting)
Gmail API (read metadata, batch delete, batch modify, send unsubscribe emails)
Gemini Flash API (free tier AI classification, 1,500 req/day)
Rule-based classification fallback (must be indistinguishable in quality from AI)
PWA (installable on mobile, add to home screen)

UI:

Neumorphism / Soft UI style throughout
Base: black charcoal (#1a1a1e)
Card surface: #222228
Neumorphic shadows: light #2c2c35, dark #111116
Accent colors are functional:
Red #e84141 — delete, destructive actions
Green #26de81 — safe senders, success
Amber #ffb142 — unsure classification, warnings
Blue #45aaf2 — unsubscribe, neutral info
Purple #a55eea — selected state
Font: Space Grotesk (Google Fonts) — used for all text
No borders — depth is created entirely through shadows
All interactive elements have subtle press animations (shadow inversion on click)

Database schema (Supabase):

-- profiles: extends auth.users

CREATE TABLE public.profiles (

  id UUID REFERENCES auth.users(id) ON DELETE CASCADE PRIMARY KEY,

  email TEXT NOT NULL,

  google_refresh_token TEXT,

  last_scan_at TIMESTAMPTZ,

  created_at TIMESTAMPTZ DEFAULT NOW()

);

-- shared classification cache across all users

CREATE TABLE public.sender_classifications (

  domain TEXT PRIMARY KEY,

  classification TEXT NOT NULL CHECK (classification IN ('junk', 'safe', 'unsure')),

  confidence FLOAT,

  method TEXT CHECK (method IN ('ai', 'rule_based')),

  reason TEXT,

  classified_at TIMESTAMPTZ DEFAULT NOW(),

  updated_at TIMESTAMPTZ DEFAULT NOW()

);

-- per-user sender scan results

CREATE TABLE public.user_senders (

  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,

  user_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE,

  sender_email TEXT NOT NULL,

  sender_name TEXT,

  domain TEXT,

  email_count INTEGER DEFAULT 0,

  unread_count INTEGER DEFAULT 0,

  has_unsubscribe_header BOOLEAN DEFAULT FALSE,

  unsubscribe_mailto TEXT,

  unsubscribe_url TEXT,

  unsubscribe_post BOOLEAN DEFAULT FALSE,

  gmail_labels TEXT[],

  classification TEXT CHECK (classification IN ('junk', 'safe', 'unsure')),

  classification_method TEXT,

  is_unsubscribed BOOLEAN DEFAULT FALSE,

  last_scanned_at TIMESTAMPTZ DEFAULT NOW(),

  UNIQUE(user_id, sender_email)

);

-- user overrides: manual safe/junk markings

CREATE TABLE public.user_sender_overrides (

  user_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE,

  sender_email TEXT NOT NULL,

  override TEXT NOT NULL CHECK (override IN ('safe', 'junk')),

  created_at TIMESTAMPTZ DEFAULT NOW(),

  PRIMARY KEY (user_id, sender_email)

);

Key behaviors:

Users select/deselect individual senders via checkboxes before any action fires
Actions never fire automatically without user selection + button press
Bulk actions show real-time progress where possible, animated text fallback otherwise
Classification: Gemini Flash first → rule-based fallback if quota exceeded
Classification results cached in Supabase — same domain never classified twice
Gmail OAuth uses sensitive scopes (gmail.modify) — unverified app 100-user cap is known and acceptable for now
Gmail only (no Outlook)

Database schema — apply the full schema from master context to Supabase. Also add Row Level Security:

ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can only access own profile" ON public.profiles FOR ALL USING (auth.uid() = id);

ALTER TABLE public.user_senders ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can only access own senders" ON public.user_senders FOR ALL USING (auth.uid() = user_id);

ALTER TABLE public.user_sender_overrides ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can only access own overrides" ON public.user_sender_overrides FOR ALL USING (auth.uid() = user_id);

-- sender_classifications is shared/read by all authenticated users

ALTER TABLE public.sender_classifications ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Authenticated users can read classifications" ON public.sender_classifications FOR SELECT USING (auth.role() = 'authenticated');

CREATE POLICY "Service role can write classifications" ON public.sender_classifications FOR ALL USING (auth.role() = 'service_role');
